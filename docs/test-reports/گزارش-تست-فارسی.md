## ۵) شواهد تست دستی (زنده)

| بررسی | شاهد |
|---|---|
| ریدایرکت `/` → `/en` | HTTP **۳۰۷** |

| API ادمین (با لاگین) | `GET /api/admin/leads` → **۲۰۰** |
| API ادمین (بدون لاگین) | `GET /api/admin/users` → **۴۰۱** |
| RBAC | editor/viewer → `/api/admin/users` → **۴۰۳** |
| خروجی CSV لیدها | **۲۰۰** `text/csv; charset=utf-8` |
| خروجی Excel لیدها | **۲۰۰** `…spreadsheetml.sheet` |

## ۸) نحوه‌ی اجرای مجدد تست‌ها

```bash
cd <repo>
npx next start -p 3100      # سرو نسخه‌ی production (ری‌استارت، بودجه‌ی rate-limit را صفر می‌کند)
npm run test:e2e            # ۴۲ تست Playwright
npm run typecheck           # بررسی نوع TypeScript
npm run lint                # ESLint
npm run build               # ساخت production

npx playwright show-report  # گزارش بصری HTML تعاملی
```

> **نکته:** پیش از هر اجرای کامل، سرور را ری‌استارت کنید تا بودجه‌ی in-memory محدودیت نرخ لید صفر شود (وگرنه تست rate-limit ممکن است وضعیت متفاوت بدهد).
